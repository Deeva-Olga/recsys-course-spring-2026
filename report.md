## Homework 2 Report

## Abstract
Исследуется легковесный session-aware reranker для музыкальных рекомендаций. Предлагается эвристический подход: кандидаты извлекаются из предобученной SasRec-I2I таблицы, затем переранжируются с учётом трёх сигналов — ранг в I2I-списке, разнообразие артистов и популярность трека. Такой подход позволяет улучшить персонализацию без обучения отдельной модели и без риска переобучения.

## Детали реализации

### Архитектура модели
Архитектура рекомендера. SessionPopReranker работает в реальном времени и не требует precomputed пользовательских профилей:

Сессия (3–5 треков) → [Извлечение] → SasRec‑I2I(N=30) →
→ [Скоринг] → (ранг + штраф артиста + бонус популярности) →
→ [Агрегация] → взвешенная сумма ($0{,}8^i$) →
→ Трек с макс. скором



Защита от ошибок. Рекомендер включает многоуровневый fallback:
1. Если история пустая → SasRec-I2I fallback
2. Если кандидатов нет → Random fallback
3. Если ошибка в Redis → логирование + fallback
4. Если pickle не читается → попытка JSON


Компонент        | Формула                              | Параметр
----------------|--------------------------------------|-----------------------------------------
Recency weight  | $0{,}8^i$                         | $i$ = позиция якоря
Rank bonus       | $\frac{1{,}0}{\text{rank} + 1}$   | $\text{rank}$ = позиция в I2I
Artist penalty   | $0{,}15 \times (\text{count}^{1{,}3})$ | $\text{count}$ = сколько раз артист в сессии
Popularity bonus | $\frac{0{,}12}{\log1p(\text{PopRank} + 1)}$ | $\text{PopRank}$ = позиция в каталоге
